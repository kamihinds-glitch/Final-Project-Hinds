Responsive CSS Cheat Sheet

Quick reminders for making layouts work on phones, tablets, and desktops.

1. Always include this meta tag

<meta name="viewport" content="width=device-width, initial-scale=1.0">

2. Use flexible sizing

width: 100%;
max-width: 600px;

3. Make images responsive

img {
  max-width: 100%;
  height: auto;
}

4. Avoid horizontal overflow

body {
  overflow-x: hidden;
}

5. Common breakpoints

@media (max-width: 1024px) { }
@media (max-width: 768px)  { }
@media (max-width: 640px)  { }

6. Stack flex items on smaller screens

@media (max-width: 768px) {
  .row {
    flex-direction: column;
  }
}

7. Make buttons easier to tap

@media (max-width: 640px) {
  .btn {
    width: 100%;
  }
}

8. Reduce heading sizes when needed

@media (max-width: 640px) {
  h1 {
    font-size: 32px;
  }
}

9. Make paragraphs easier to read

@media (max-width: 640px) {
  p {
    font-size: 14px;
    line-height: 1.6;
  }
}

10. Check sections for spacing

@media (max-width: 640px) {
  section {
    padding: 0 16px;
  }
}

11. Stack cards/grid items

@media (max-width: 768px) {
  .cards {
    grid-template-columns: 1fr;
  }
}

12. Quick debugging checklist

  • Does anything overflow sideways?
  • Can you read the text without zooming?
  • Do buttons still fit?
  • Does the nav still work?
  • Are images shrinking correctly?
  • Does the layout stack nicely?

Fast rule of thumb

If a section looks bad on mobile, usually one of these is the problem:

  • wrong flex-direction
  • fixed width or margin that is too big
  • text too large
  • button/input too wide
  • image not shrinking
